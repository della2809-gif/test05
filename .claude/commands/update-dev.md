프로젝트의 개발 업데이트 문서를 작성/정리한다.

반드시 아래 순서로 진행한다.

1. 기존 업데이트 문서 양식을 먼저 확인한다.
   - `04_docs/260525_업데이트.md`
   - `04_docs/260527_업데이트.md`
   - 필요하면 다른 `04_docs/*_업데이트.md` 파일도 확인한다.

2. 오늘 날짜 기준으로 파일을 작성한다.
   - 경로: `04_docs/YYMMDD_업데이트.md`
   - 예: 2026년 6월 1일이면 `04_docs/260601_업데이트.md`

3. 실제 변경 사항을 확인한다.
   - `git status --short`
   - `git diff --name-status`
   - 필요한 경우 관련 파일 diff/read
   - 모르는 변경 사항은 추측하지 않는다.

4. 문서 양식은 기존 `04_docs` 스타일을 따른다.
   - 상단 제목
   - 세션 한 줄 요약
   - `## 변경된 파일 목록` 표
   - 각 주요 변경 항목마다:
     - `**왜 만들었나**`
     - `**뭘 할 수 있나**`
   - `## 검수 결과`
   - `## 기술 변경 요약`

5. 문체 규칙
   - 한국어로 작성한다.
   - 비개발자/클라이언트도 이해할 수 있게 쓴다.
   - “컴포넌트”, “마이그레이션”, “스키마” 같은 개발 용어는 필요할 때만 쓰고, 가능하면 쉬운 설명을 함께 적는다.
   - 단순히 코드 파일만 나열하지 말고, 사용자 입장에서 무엇이 달라졌는지 설명한다.

6. 작성 후 반드시 파일을 다시 읽고 확인한다.

사용자가 인자를 주면 그 내용을 이번 업데이트의 세션/작업 범위로 우선 반영한다.

인자: $ARGUMENTS
