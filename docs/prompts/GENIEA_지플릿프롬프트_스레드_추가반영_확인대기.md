# GENIEA 지플릿 프롬프트 스레드 - 추가 반영 / 확인 대기 정리

## 문서 목적
이 문서는 `지플릿 프롬프트 제작` 스레드에서 나온 추가 반영 후보와 확인 필요 내용을 누적 정리하기 위한 작업 문서다.

- 실제 DB/코드/운영자페이지에 즉시 반영된 내용이 아님.
- 매니저 확인이 필요한 항목은 일괄 요청 전까지 `확인 필요`로 둔다.
- 지플릿 프롬프트 작성 전 먼저 확정해야 하는 공통 구조와, DB 추가 반영 후보를 분리해 관리한다.

---

## 1. 지플릿 전체 공통 구조 확인 필요

### 1-1. 셀프 모드 / 가이드 모드

#### 현재 정리
지플릿 전체는 특정 지플릿 1개만의 문제가 아니라, 모든 지플릿이 아래 두 가지 사용 목적을 가질 수 있다.

- 셀프 모드: 사용자 본인의 기록, 목표, 상담 준비, 미팅 피드백, 실행계획을 정리
- 가이드 모드: 사용자가 만나는 고객, 소비자, 파트너, 팀원을 상담/안내하기 위한 질문 흐름과 말버전 정리

#### 확인 필요
- 셀프/가이드 모드 선택 위치가 지플릿 실행 화면인지, 케이스관리의 AI 정리 기능인지, 둘 다인지 확인 필요
- 선택값이 시스템 프롬프트에 전달되는 구조인지 확인 필요
- 지플릿별 시작 프롬프트에서 매번 물어볼지, 공통 UI 선택값으로 받을지 확인 필요

#### 매니저 일괄 요청 문구 후보
`셀프/가이드 모드 선택이 지플릿 실행 화면의 공통 선택값인지, 케이스관리 AI 정리 기능의 선택값인지, 또는 둘 다에 적용되는 값인지 확인 부탁드립니다. 이 기준이 정리되어야 24개 지플릿 시스템 프롬프트에서 셀프/가이드 출력 방식을 일관되게 반영할 수 있습니다.`

#### 상태
확인 필요

---

### 1-2. 성향 선택: 논리적 / 감성적 / 실용적

#### 현재 정리
메일 기준으로 AI 정리 기능은 성향 선택에 따라 출력 톤이 구분되어야 한다.

- 논리적: 기준, 흐름, 우선순위, 판단 근거 중심
- 감성적: 마음, 동기, 고민, 기대를 따뜻하게 정리
- 실용적: 바로 실행할 수 있는 행동, 문장, 체크리스트, 다음 단계 중심

#### 확인 필요
- 성향 선택이 지플릿 전체 공통값인지, 케이스관리 AI 정리용인지 확인 필요
- 선택값이 실제 답변 생성에 전달되는지 확인 필요
- 선택하지 않았을 때 기본값을 `따뜻하지만 실용적인 톤`으로 둘지 확인 필요

#### 매니저 일괄 요청 문구 후보
`논리적/감성적/실용적 성향 선택이 어느 화면의 어떤 기능에 붙어 있는지 확인 부탁드립니다. 지플릿 프롬프트에서는 이 선택값을 기준으로 같은 지플릿도 출력 톤이 달라지도록 설계할 예정입니다.`

#### 상태
확인 필요

#### 확인 필요

- 내가 지금 어떤 단계인지

- 무엇을 준비해야 하는지
- 미팅 전/중/후 무엇을 해야 하는지
- 내 단계에서 어떤 지플릿을 써야 하는지

`케이스관리 = 내가 지금 사업을 어떻게 진행할지 잡아주는 단계별 코칭 흐름`

## 3. 회원등록 지플릿 / 회원관리 차이 확인 필요

### 3-1. 회원관리

#### 현재 화면 기준 정리
회원관리는 실제 회원 데이터를 저장하고 관리하는 기능으로 보인다.

확인된 관리 항목:
- 이름
- 연락처
- 허브 회원번호
- 가입일
- 첫 주문일
- AO 예정일
- 케어 모드
- 접촉 주기
- 쿠폰 잔여
- 메모
- 마일스톤 / 보너스 일정

#### 역할 후보
`회원관리 = 등록된 회원을 저장하고 AO, 쿠폰, 접촉주기, 후속관리 일정을 관리하는 실제 관리 화면`

---

### 3-2. 회원등록 지플릿

#### 현재 화면 기준 정리
회원등록 지플릿은 허브 캡처나 회원 정보를 입력받아 회원관리 입력값으로 정리하는 AI 도우미로 보인다.

역할 후보:
- 허브 캡처/텍스트에서 회원 정보 추출
- 회원관리 입력폼 순서대로 정리
- 누락값은 확인 필요 표시
- 회원등록 절차 안내
- 신규 회원에게 보낼 첫 안내/후속관리 문장 작성

#### 확인 필요
- 회원등록 지플릿에서 정리한 값이 회원관리로 자동 저장/연동되는지
- 자동 연동이 아니라면 유저가 같은 정보를 두 번 입력해야 하는 구조인지
- 최소한 회원등록 지플릿 출력 형식을 회원관리 입력폼 순서와 동일하게 맞추면 되는지
- 회원관리의 일정 연동 기준이 가입일/첫 주문일/AO 예정일 중 무엇인지

#### 매니저 일괄 요청 문구 후보
`메인페이지 회원관리는 실제 회원 저장/일정 관리 기능이고, 지플릿의 회원등록은 허브 캡처나 회원 정보를 AI가 정리해 회원관리 입력값으로 만드는 앞단 도우미로 이해했습니다. 회원등록 지플릿 결과가 회원관리로 자동 연동되는지, 아니면 출력 형식만 맞춰 수동 입력하는 구조인지 확인 부탁드립니다.`

#### 상태
확인 필요

## 5. DB 추가 반영 후보: 시중 영양제 질문 대응 기준

### 5-1. 반영 후보 요약
사용자가 TV 광고 영양제, 시중 제품, 타사 영양제와 유사나 차이를 물을 때의 답변 기준.

#### 핵심 방향
- 시중 제품 답부터 하지 않기
- 서플리먼트 안내 먼저 하기
- 비교 평가 기준 제공
- 성분/함량 이전에 포뮬러 보기
- 흡수율 보기
- 실제 라벨 성분·함량 검증 보기
- 안정성/독성 보기
- 영양제는 쇼핑 아이템이 아니라 기본에 충실한 세포영양으로 설명

#### DB 후보 분류
- Script DB 후보
- FAQ DB 후보
- GenieA 답변 기준 후보
- Reference DB / Link DB 근거자료 연결 후보

#### 상태
DB 반영 후보 준비됨 / 실제 업로드 전 확인 필요

---

### 5-2. Script DB 후보

#### 자료명
시중 영양제와 유사나 차이 질문 대응 말버전

#### 사용 상황
사용자가 TV 광고 영양제, 홈쇼핑 영양제, 약국 영양제, 시중 제품, 타사 영양제와 유사나를 단순 가격/함량/인지도로 비교할 때

#### 실제 말버전 요약
시중 영양제도 좋은 제품이 있을 수 있다. 다만 영양제는 성분표에 무엇이 들어있는지, 함량이 높은지만 보고 고르기에는 부족하다. 같은 성분명이라도 원료, 포뮬러, 배합, 흡수율, 몸 안에서의 활용, 라벨 검증, 안전성, 독성 관리, 제조 품질이 다를 수 있다. 유사나는 단순 쇼핑용 영양제가 아니라 기본에 충실한 세포영양 제품이라는 기준으로 설명한다.

#### 상태
표현순화 후 업로드가능 / 근거자료 연결 시 더 좋음

---

### 5-3. FAQ DB 후보

#### 질문
시중 영양제나 타사 영양제와 유사나는 뭐가 다른가요?

#### 답변 방향
성분명과 함량만으로 판단하지 않고, 원료 품질, 포뮬러, 흡수율, 라벨 검증, 안전성, 독성 관리, 제조 품질, 세포영양 관점으로 비교 기준을 잡아준다.

#### 상태
표현순화 후 업로드가능 / 근거자료 연결 시 더 좋음

---

### 5-4. FAQ DB 추가 후보

#### 질문
영양제를 고를 때 왜 성분표나 광고만 보면 안 되나요?

#### 답변 방향
영양제는 쇼핑 아이템이 아니라 내 몸에 들어가는 세포영양 제품이므로, 광고/인지도/함량만이 아니라 실제 성분 검증, 제조 품질, 안전성, 독성, 원료 품질을 함께 봐야 한다고 설명한다.

#### 상태
확인필요 / 근거자료 연결 후 업로드가능

---

### 5-5. Reference DB / Link DB 근거자료 후보

#### 연결 후보
- 뉴트리셔널 써플리먼츠 자료
- 의사·약사 처방전 편람 등재 관련 자료
- 제조시설 및 품질관리 기준 확인 자료
- 유사나 특허기술 / 인셀리전스 / 포뮬러 관련 공식 자료
- 라벨 성분·함량 검증 관련 공식 자료
- 안전성·독성 관리 관련 공식 자료
- 건강기능식품 신고제/표시·광고/사후관리 관련 공식 자료

#### 확인 필요
- 홍삼/백수오/노니가루 사례는 뉴스 사례로 사용할 경우 실제 기사 또는 공식 발표 자료 확인 필요
- 건강기능식품 신고제 및 사전검열 여부는 식약처/공식 제도 자료 확인 필요
- 유사나 제조시설, 특허기술, 처방전 등재 근거는 공식자료 연결 필요

#### 상태
확인필요
