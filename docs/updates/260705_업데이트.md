## 코칭 지플릿 프롬프트 개선

**왜 만들었나**
꿈과목표·결심결단 같은 코칭형 지플릿이 질문을 깊게 이어가지 못하고 "1) 상담 판단 2) 추천 방향..." 식 보고서 형식으로 성급하게 정리해버리는 문제가 있었음. 원인은 자유채팅용 "직원 상담 4단 형식" 지시가 코드에서 모든 지플릿에 강제 주입되어 운영자가 쓴 프롬프트와 충돌한 것임. 운영자 프롬프트 수정만으로는 해결 불가능한 구조였음.

**뭘 할 수 있나**
1. **깊은 질문 흐름** — 코칭형 지플릿(계산·검색 기능이 없는 커스텀 지플릿)은 4단 형식 없이 질문을 한 번에 1~2개씩, 여러 턴에 걸쳐 깊게 이어감. 사용자가 실행 계획만 던져도 이유(WHY)를 먼저 되물음.
2. **성급한 정리 금지** — 사용자가 요청하거나 충분한 대화가 쌓이기 전에는 요약·정리로 건너뛰지 않음.
3. **금지어 자동 적용** — 접근·컨택·공략·설득 등 대표님이 지정한 부정적 표현을 쓰지 않고 존중 표현으로 대체함.
4. **기존 동작 보존** — 자유채팅·수당계산·제품검색 등 시스템 지플릿은 기존 4단 형식 그대로 유지됨. 실서버에서 코칭/자유채팅 양쪽 모두 확인됨.

## 관리자 레퍼런스 파일 본문 미리보기

**왜 만들었나**
관리자 페이지에서 파일명만 보이고 내용을 열람할 수 없어, 클라이언트가 "자료가 DB에 제대로 들어갔는지" 검수할 방법이 없었음. 처방전 자료처럼 반영 안 된 파일을 확인하지 못해 반복 문의가 발생했음.

**뭘 할 수 있나**
1. **본문 미리보기** — 관리자 페이지 → 레퍼런스 파일에서 파일을 클릭하면 추출된 본문·조각(청크) 수·업로드 정보를 볼 수 있음.
2. **미반영 경고** — 본문이 추출되지 않은 파일(스캔본 PDF 등)은 "AI 검색에 반영되지 않음" 경고가 표시되어 즉시 식별 가능함.
3. **원본 다운로드** — 업로드한 원본 파일을 바로 내려받을 수 있음.

## 레퍼런스 한글 검색 키워드

**왜 만들었나**
영문 자료(예: Nutritional Supplements 원서)는 본문이 영어라 한글 질문으로 검색되지 않는 문제가 있었음. 제품 DB에는 별칭 검색이 있었으나 레퍼런스 파일에는 없었음.

**뭘 할 수 있나**
1. **키워드 등록** — 파일 상세 화면에서 한글 검색 키워드(예: 암웨이, 센트룸, 영양제 등급)를 등록하면 그 키워드가 포함된 한글 질문에 해당 자료가 답변 근거로 사용됨.
2. **적용 완료** — 같은 날 후반 승인으로 라이브 DB에 마이그레이션 적용됨. 키워드 저장이 바로 사용 가능하며, 처방전 2건에 키워드가 실제 등록·검색 확인됨.

## 자동견적 — 말로 물으면 패키지 견적 (신규)

**왜 만들었나**
자동견적 지플릿이 건강 체크리스트 사진 분석만 가능하고, "리셋 2주 견적 내줘" 같은 말로 하는 질문에는 일반 답변만 하던 상태였음. 클라이언트 1차 검수 항목이자 7/3 통화에서 확인된 핵심 기대 기능. 패키지 구성품 데이터가 계산에 전혀 쓰이지 않던 구조도 함께 해결이 필요했음.

**뭘 할 수 있나**
1. **말로 견적 요청** — 자동견적 지플릿에 "리셋 해독 2주 패키지 견적 내줘", "50만원 이하 다이어트 패키지 추천"처럼 입력하면 패키지 추천 카드(최대 3개)가 나옴. 구성품별 수량·단가, 패키지 가격, 점수, 견적 문구 복사 버튼 포함. 실서버 3개 시나리오 검증 완료.
2. **예산 필터** — "50만원 이하", "100만원 정도"처럼 예산을 말하면 그 범위에서 추천함.
3. **가격을 지어내지 않음** — AI가 고르는 방식이 아니라 규칙 기반 계산이라, 제품 DB에서 단가를 못 찾은 구성품은 "단가 미확인"으로 정직하게 표시됨.
4. **기존 기능 유지** — 사진 업로드 시 건강 분석 견적은 기존 그대로 동작함.

## 패키지 구성품 데이터 정리 + 관리자 입력칸 (신규)

**왜 만들었나**
패키지 47개 중 30개의 구성품이 설명글 안에만 있어 자동견적 계산에 쓸 수 없었고, 관리자 화면에 구성품 입력칸 자체가 없어 클라이언트가 새 패키지를 만들어도 자동견적에 반영될 수 없는 구조였음.

**뭘 할 수 있나**
1. **47개 전체 구조화 완료** — 설명글의 구성품을 파싱해 전 패키지가 제품명+수량 데이터를 보유함 (전체 백업 후 진행).
2. **구성품 입력칸** — 관리자 패키지 수정 화면에서 구성품을 행 단위(제품명 자동완성+수량)로 추가/삭제 가능함.
3. **설명글에서 불러오기** — 기존 습관대로 설명글에 "헬스팩 1통…"을 붙여넣고 버튼 한 번이면 구성품 행이 자동으로 채워짐.
4. **구성품 없음 경고** — 목록에서 구성품이 빈 패키지는 경고가 표시되어 자동견적 미반영 상태를 바로 알 수 있음.
