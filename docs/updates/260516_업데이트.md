## 저장 버튼 먹통 오류 수정

**왜 만들었나**
회원 정보 수정 화면에서 주문 주기(AO 날짜)를 바꾼 뒤 저장 버튼을 눌렀을 때 아무 반응이 없는 현상이 보고됨. 네트워크 오류 발생 시 저장 버튼이 영구적으로 비활성 상태로 굳어버리는 구조적 결함이 원인이었음. 실무에서 자주 사용하는 기능인 만큼 즉시 수정이 필요함.

**뭘 할 수 있나**
1. **저장 버튼 정상화** — 이제 저장 중 오류가 발생해도 버튼이 다시 활성화되어 재시도할 수 있음
2. **오류 메시지 표시** — 저장에 실패하면 "저장 중 오류가 발생했습니다"라는 안내가 화면에 표시됨

## 회원 등록 필수값 강화

**왜 만들었나**
기존에는 이름만 입력하면 회원 등록이 가능해서 연락처나 회원번호 없이 등록된 회원이 생기는 문제가 있었음. 이름·연락처·회원번호가 모두 있어야 이후 관리(AO 확인, 미팅 준비 등)가 의미 있으므로 세 항목을 모두 필수로 지정함.

**뭘 할 수 있나**
1. **필수 항목 안내** — 연락처·허브 회원번호 입력란에 * 표시가 추가됨
2. **빈 값 차단** — 세 항목 중 하나라도 비어 있으면 다음 단계로 넘어가지 않고 오류 메시지가 표시됨

## AO·보너스 날짜 기준 분리

**왜 만들었나**
자동 주문(AO) 날짜와 보너스·마일스톤 날짜가 동일한 기준(첫 주문일)으로 계산되어 실제 정책과 다른 날짜가 나오는 문제가 있었음. 회의에서 확인한 정책은 "보너스는 가입일 기준, AO는 주문일 기준"이므로 계산 로직을 정책에 맞게 수정함.

**뭘 할 수 있나**
1. **보너스·마일스톤 날짜** — 가입일로부터 4주·8주·13주·17주 기준으로 계산됨 (쿠폰 사용 가능일 포함)
2. **AO 예정일** — 첫 주문일 기준으로 계산됨 (기존과 동일한 5주차 일요일)
3. **소급 입력 지원** — 과거 날짜로 등록해도 날짜 계산이 정상 작동함

## 토요일 가입 매출 주차 체크박스 추가

**왜 만들었나**
USANA 정책상 토요일 오후 3시가 매출 마감 기준이라 같은 토요일이라도 3시 전/후에 따라 이번 주 또는 다음 주 매출로 분류됨. 시스템이 날짜만 저장하고 시간은 저장하지 않아 판단이 불가능했으므로 관리자가 직접 선택할 수 있는 체크박스를 추가함.

**뭘 할 수 있나**
1. **가입일 체크박스** — 가입일이 토요일이면 "오후 3시 이후에 가입하셨나요?" 항목이 표시됨. 체크하면 다음 주 1주차 시작으로 계산됨
2. **주문일 체크박스** — 첫 주문일이 토요일이면 동일한 체크박스가 표시되어 AO 주기 시작 시점에 반영됨

## FAQ·링크 카테고리 폴더 필터

**왜 만들었나**
FAQ와 링크 데이터가 많아질수록 목록 전체를 스크롤해야 하는 불편함이 예상됨. 카테고리별로 묶어 보고 원하는 분류만 빠르게 조회할 수 있는 구조가 필요함.

**뭘 할 수 있나**
1. **카테고리 버튼 필터** — FAQ 관리·링크 관리 화면 상단에 카테고리 버튼이 표시됨. 버튼 클릭 시 해당 카테고리 항목만 보임
2. **전체 보기** — "전체" 버튼으로 모든 항목으로 돌아올 수 있음
3. **자동 목록 생성** — 데이터에 카테고리가 입력되면 버튼이 자동으로 생성되어 별도 설정 불필요함

## 채팅 입력창 엔터키 동작 변경

**왜 만들었나**
모바일에서 채팅 메시지를 여러 줄로 입력하다가 엔터키를 누르면 의도치 않게 전송이 되는 문제가 있었음. 긴 내용을 입력할 때 줄바꿈 후 계속 이어서 쓸 수 없어 불편하다는 피드백이 있었음.

**뭘 할 수 있나**
1. **엔터 = 줄바꿈** — 이제 엔터키를 누르면 줄이 바뀌고 전송되지 않음 (모바일·PC 동일)
2. **버튼으로만 전송** — 메시지를 보내려면 오른쪽 전송 버튼을 눌러야 함
3. **단축키 전송** — PC에서 빠르게 보내려면 Ctrl+Enter(윈도우) 또는 Cmd+Enter(맥)를 사용할 수 있음

## 미팅 준비 프롬프트 탭 추가

**왜 만들었나**
미팅 준비 시나리오에서 AI가 고객 정보를 하나씩 순서대로 질문하는 방식만 있어서, 여러 정보를 한 번에 입력하고 싶은 경우에는 대응이 안 됐음. 어드민에서 미팅 준비 전용 프롬프트를 설정할 수 있도록 탭을 별도로 추가함.

**뭘 할 수 있나**
1. **미팅준비 탭** — 어드민 프롬프트 설정 화면에 "미팅준비" 탭이 추가됨
2. **일괄 입력 허용** — 이 탭에 "이름, 나이, 건강목표 등을 한 번에 입력해도 됩니다" 식의 프롬프트를 작성하면 AI가 일괄 입력을 인식하도록 동작함
3. **작성 가이드** — 탭 내부에 일괄 입력 지원 프롬프트 작성 팁이 안내 박스로 표시됨

3. **즉시 반영** — 저장하면 목록의 제목이 바로 변경됨

## 마일스톤(쿠폰·주차) 날짜 기준 수정

**왜 만들었나**
쿠폰 4주차·8주차, 13주차·17주차 도달 날짜가 USANA 영업 주차 기준이 아닌 가입일에 단순히 날짜를 더하는 방식으로 잘못 계산되고 있었음. 수요일에 가입한 회원도 그 주 토요일 오후 3시를 1주차 시작 기준점으로 삼아야 하는데, 기존 코드는 이 주차 경계를 무시하고 가입일 그대로 더했음. AO(자동주문) 날짜는 이미 올바르게 주차 경계를 반영하고 있어서, 마일스톤도 동일한 방식으로 수정함.

**뭘 할 수 있나**
1. **쿠폰 날짜 정확화** — 수요일 가입이든 금요일 가입이든 그 주 토요일 오후 3시(1주차 시작점)로부터 4주·8주 후가 쿠폰 사용 가능일로 표시됨
2. **13주·17주차 날짜 정확화** — 동일한 방식으로 해당 주차 시작일이 올바르게 계산됨
3. **토요일 체크박스 연동** — 가입일이 토요일이고 오후 3시 이후 체크 시, 다음 주 토요일이 1주차 시작점이 되어 마일스톤 날짜도 1주 늦춰짐
