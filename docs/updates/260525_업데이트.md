## 한글 입력 마지막 글자 미저장 버그 수정 (BUG-B01, BUG-B02)

**왜 만들었나**
DB 항목을 추가하거나 수정할 때 제목란에 한글을 입력하면 마지막 글자 1개가 저장되지 않는 문제가 발생함. 카테고리 입력창에서도 동일한 현상이 있었음. 데이터를 두 번 입력해야 정상 저장되는 불편함이 있어 수정이 필요함.

**뭘 할 수 있나**
1. **제목 입력** — 한글 입력 후 저장 시 마지막 글자까지 정상 저장됨
2. **카테고리 입력** — 스토리 DB 등 카테고리 입력창에서도 동일하게 정상 저장됨

## 패키지 DB 저장 오류 수정 (BUG-B03)

**왜 만들었나**
패키지 DB에서 항목을 추가하려고 하면 "패키지 필드 정보를 입력하세요.나." 오류가 발생하며 저장이 되지 않았음. 이로 인해 이희정님이 패키지 데이터를 한 건도 입력하지 못한 상태였음.

**뭘 할 수 있나**
1. **패키지 항목 추가** — 오류 없이 정상적으로 저장됨
2. **데이터 입력 재개** — 막혀 있던 패키지 DB 데이터 입력 가능함

## AI 채팅 응답 실패 오류 수정 (BUG-B04)

**왜 만들었나**
"프로그래머 출신 유사나 성공자 사례는?", "남매가 유사나 함께 하는 이야기는?" 등 성공사례 관련 질문 시 "AI 응답 생성에 실패했습니다" 오류가 반복 발생함. 스토리 DB에 저장된 내용이 너무 길어 AI가 한 번에 처리하지 못하는 구조적 문제였음.

**뭘 할 수 있나**
1. **성공사례 질문** — 오류 없이 정상적으로 AI 응답이 출력됨
2. **다양한 질문** — 스토리 관련 키워드가 포함된 질문에서 안정적인 응답 가능함

## 패키지 DB 구조 변경 — Purpose 제거, 태그 추가 (C-01, C-02)

**왜 만들었나**
패키지 DB에 Purpose(용도) 항목이 있었으나 패키지는 종류가 다양해 Purpose로 구분하는 것이 적합하지 않다는 피드백이 있었음. 대신 카테고리와 태그로 분류하는 것이 더 효율적임.

**뭘 할 수 있나**
1. **카테고리 분류** — Purpose 항목 없이 카테고리로만 패키지를 구분함
2. **태그 입력** — 각 패키지에 태그를 입력해 세부 분류 및 검색 활용 가능함

## 링크 DB 설명 명확화 (C-03)

**왜 만들었나**
링크 DB에 유튜브 링크와 일반 링크가 혼재되어 어느 DB에 어떤 링크를 저장해야 하는지 불명확했음.

**뭘 할 수 있나**
1. **링크 DB 용도 안내** — 인스타그램·카카오채널·기사·웹사이트 등 인터넷 링크 전용임이 화면에 표시됨
2. **유튜브 분리** — 유튜브 영상은 유튜브 DB에 따로 입력하는 구조로 명확히 구분됨

## 제품 DB — 유사IQ 링크 추가 및 AI 채팅 연결 (D-01)

**왜 만들었나**
AI 채팅에서 제품을 추천할 때 텍스트 설명만 나오고 실제 구매 링크가 함께 제공되지 않았음. 고객이 제품 설명을 보고 바로 이동할 수 있는 링크가 필요했음.

**뭘 할 수 있나**
1. **유사IQ 링크 입력** — 제품 관리 화면에서 각 제품의 유사IQ 링크를 등록 가능함
2. **채팅 응답에 링크 포함** — AI가 제품을 추천할 때 설명과 함께 유사IQ 링크도 자동 출력됨

## 레퍼런스 파일 PDF 업로드 수정

**왜 만들었나**
레퍼런스 파일 메뉴에서 PDF를 업로드하면 응답 없이 멈추다가 실패하는 문제가 있었음. PDF 내용을 분석하는 과정이 서버 응답을 통째로 막는 구조적 문제였음.

**뭘 할 수 있나**
1. **PDF 업로드** — PDF 파일을 정상적으로 업로드 가능함
2. **빠른 응답** — 파일 업로드 완료 즉시 성공 메시지가 표시되며, 내용 분석은 이후 자동 처리됨

## 스토리 DB 저장 오류 수정

**왜 만들었나**
스토리 DB에서 `health`, `business`, `combined` 이외의 카테고리를 입력하면 저장이 되지 않는 문제가 있었음. 데이터베이스 내부에 카테고리 값을 제한하는 규칙이 남아 있어 자유로운 카테고리 입력이 막혀 있었음.

**뭘 할 수 있나**
1. **카테고리 자유 입력** — 어떤 카테고리 이름을 입력해도 정상적으로 저장됨
2. **기존 데이터 유지** — 기존에 저장된 항목에는 영향 없음

## 지플릿 관리 — 시스템 지플릿 잠금 해제

**왜 만들었나**
기존에 기본 제공되는 지플릿(수당 계산, 제품 추천 등)은 이름·참조 DB·활성화 여부 등 대부분의 항목이 수정 불가 상태였음. 관리자가 운영 중 내용을 조정하거나 비활성화할 필요가 생겨 모든 필드를 수정 가능하도록 변경함.

**뭘 할 수 있나**
1. **모든 필드 수정** — 이름, 설명, 태그, 색상, 시작 메시지, 참조 DB, 케이스 연결, 시스템 프롬프트를 자유롭게 수정 가능함
2. **활성화 토글** — 기본 제공 지플릿도 켜고 끌 수 있음
3. **삭제** — 필요 시 기본 제공 지플릿도 삭제 가능함
