## 미팅 준비 / 신규 제품 상담 — 질문 방식 개선

**왜 만들었나**
기존에는 AI가 상담 대상자 정보를 한 가지씩 차례로 물어보는 방식이었음. 예를 들어 이름을 물어보고, 답하면 나이를 물어보고, 또 답하면 상황을 물어보는 식이었음. 대화가 길어지고 번거롭다는 피드백이 있었음. 필요한 정보를 한 번에 확인하고 빠지는 것만 추가로 물어보는 방식으로 변경 요청을 반영함.

**뭘 할 수 있나**
1. **한 번에 질문** — 미팅 준비 또는 신규 제품 상담을 시작하면, AI가 필요한 정보 항목 전체를 번호 목록으로 한 번에 질문함
2. **빠진 항목만 추가 질문** — 사용자가 답변한 내용에서 파악할 수 있는 항목은 자동으로 수집하고, 누락된 항목이 있는 경우에만 추가로 한 번 더 모아서 질문함
3. **불필요한 반복 없음** — 이미 답한 내용을 다시 묻거나 항목을 나눠서 반복 질문하지 않음

## AI 채팅 — 이미지 자료 자동 표시

**왜 만들었나**
어드민에서 제품 사진이나 자료 이미지를 등록해도, 기존에는 AI가 채팅 응답에서 이미지를 보여줄 방법이 없었음. 텍스트로만 설명하는 방식에 한계가 있었음. 이미지 자료와 연관된 질문이 들어오면 AI가 직접 이미지를 꺼내 보여줄 수 있도록 구현함.

**뭘 할 수 있나**
1. **이미지 자동 삽입** — 사용자가 특정 제품이나 자료를 물어보면, AI가 관련 이미지를 채팅 응답 안에 직접 표시함
2. **클릭 확대 보기** — 응답에 나타난 이미지를 클릭하면 전체 화면으로 확대해서 볼 수 있음
3. **텍스트 + 이미지 동시 제공** — 이미지만 나오는 것이 아니라, 설명 텍스트와 함께 이미지가 자연스럽게 함께 표시됨

## AI 채팅 — 응답 실시간 표시 (스트리밍)

**왜 만들었나**
기존에는 AI가 답변을 전부 완성한 뒤에야 화면에 한 번에 나타났음. 답변이 길거나 시간이 걸릴 경우 화면이 멈춰 있는 것처럼 보여 사용자가 불안함을 느낄 수 있었음. 응답이 생성되는 즉시 화면에 나타나도록 개선함.

**뭘 할 수 있나**
1. **실시간 글자 표시** — AI가 응답을 생성하는 순간부터 글자가 하나씩 이어서 나타남. 완성될 때까지 기다릴 필요가 없음
2. **첫 대화 즉시 진입** — 새 대화를 시작할 때 AI 응답을 기다렸다가 이동하는 과정이 제거됨. 전송하는 즉시 대화 화면으로 이동하고 응답이 스트리밍으로 이어짐

## 회원 등록 — 신규회원 / 기존회원 모드 분리

**왜 만들었나**
기존에는 회원 등록 화면이 신규회원 기준으로만 구성되어 있었음. 이미 활동 중인 기존회원을 시스템에 추가로 등록할 때도 동일한 4단계 플로우를 거쳐야 해서 불필요한 절차가 많았음. 신규회원과 기존회원을 처음부터 구분해서 등록할 수 있도록 분리함.

**뭘 할 수 있나**
1. **모드 선택** — 회원 등록 화면 상단에서 '신규회원' 또는 '기존회원'을 선택할 수 있음
2. **기존회원 간편 등록** — 기존회원 모드 선택 시 이름·연락처·메모 등 기본 정보만 입력하면 바로 등록됨. 4단계 플로우 없이 단순하게 처리됨
3. **신규회원 기존 플로우 유지** — 신규회원은 기존과 동일하게 4단계 순서대로 진행됨

## 회원 등록 — 주문 여부 선택지 세분화

**왜 만들었나**
기존에는 회원 등록 시 주문 여부를 '주문했음 / 아직 안 함' 두 가지로만 선택할 수 있었음. 현실적으로 "이번 주는 아니고 다음 주 할 예정"이거나 "일단 나중에 입력하겠다"는 상황이 많았음. 실제 운영 상황에 맞게 선택지를 세분화함.

**뭘 할 수 있나**
1. **4가지 선택지 제공** — 주문완료 / 아직 전 / 다음 주 예정 / 나중에 입력 중 하나를 선택할 수 있음
2. **상황에 맞는 상태 자동 설정** — 선택한 주문 상황에 따라 회원 상태(섭취중·주문대기 등)가 자동으로 맞게 설정됨

## 회원 등록·수정 — 토요일 컷오프 정책 반영

**왜 만들었나**
유사나 USANA 정책상 토요일 오후 3시(컷오프 시각) 이전과 이후 가입·주문은 적용되는 주차가 달라짐. 기존 시스템에서는 이 구분을 반영하지 못해 AO 주기나 마일스톤 날짜가 잘못 계산되는 경우가 있었음. 토요일 가입·주문의 경우 컷오프 시각 여부를 직접 선택하면 계산에 자동 반영되도록 구현함.

**뭘 할 수 있나**
1. **컷오프 체크박스 자동 표시** — 가입일 또는 첫 주문일을 토요일로 입력하면, "오후 3시 이후인가요?" 체크박스가 자동으로 나타남
2. **AO 주기 정확 계산** — 체크 여부에 따라 AO 예정일이 당주 기준 또는 다음 주 기준으로 정확하게 계산됨
3. **마일스톤 일정 자동 반영** — 컷오프 여부가 쿠폰 지급일, 13주 매칭보너스, 17주 유예마감 일정 계산에도 동일하게 반영됨
4. **회원 수정에도 동일 적용** — 기존 회원의 날짜를 수정할 때도 동일한 체크박스가 표시되어 수정 후 재계산에 반영됨

## 회원 수정 — 날짜 변경 시 일정 자동 재계산

**왜 만들었나**
기존에는 회원 수정 화면에서 가입일이나 첫 주문일을 바꿔도, 그에 따라 파생되는 마일스톤 일정이나 팔로업 일정이 자동으로 갱신되지 않았음. 날짜를 수정하면 그에 맞는 일정도 함께 다시 계산되어야 정확한 케어가 가능함.

**뭘 할 수 있나**
1. **가입일 변경 → 마일스톤 재계산** — 가입일을 수정하면 쿠폰 지급일, 13주 매칭보너스, 17주 유예마감 등 마일스톤 일정 전체가 새로 계산되어 저장됨
2. **첫 주문일 변경 → 팔로업 재생성** — 첫 주문일을 수정하면 그에 연동된 팔로업 일정이 새로 생성됨
3. **별도 작업 불필요** — 날짜만 입력하고 저장하면 나머지 계산은 시스템이 자동으로 처리함

## 회원 상세 — AO 예정일 이력 관리

**왜 만들었나**
AO 예정일은 자동 계산되기도 하고, 담당자가 상황에 따라 수동으로 바꾸기도 함. 기존에는 날짜가 바뀌어도 어떤 경위로 바뀌었는지 기록이 남지 않아 추적이 어려웠음. 자동 계산인지 수동 변경인지 구분하고, 변경 이력을 남길 수 있도록 추가함.

**뭘 할 수 있나**
1. **자동/수동 구분 배지 표시** — AO 예정일 옆에 자동 계산이면 녹색 '자동', 수동 변경이면 앰버색 '수동' 배지가 표시됨
2. **변경 이력 확인** — 수동으로 AO 날짜를 변경한 경우, 언제 바꿨는지·이전 날짜와 바뀐 날짜·메모가 이력으로 기록되고 상세 페이지에서 확인할 수 있음

## 어드민 — 이미지 자료 등록 기능 개편

**왜 만들었나**
기존 이미지 DB는 외부 URL을 직접 붙여넣는 방식이었음. 파일을 직접 올리는 것이 불가능했고, AI가 이미지를 검색해서 활용하는 구조도 연결되어 있지 않았음. 실제 파일 업로드 방식으로 전환하고, 등록한 이미지를 AI가 채팅에서 꺼내 쓸 수 있도록 연결함.

**뭘 할 수 있나**
1. **파일 직접 업로드** — 이미지 파일을 직접 업로드하면 자동으로 저장되고 URL이 생성됨
2. **스크립트 입력** — 이미지에 대한 설명 스크립트(언제 쓰는 자료인지, 어떤 내용인지)를 함께 입력할 수 있음
3. **AI 연동 자동 처리** — 등록과 동시에 AI 검색용 데이터(임베딩)가 자동 생성되어, 이후 사용자가 채팅에서 관련 내용을 물어보면 AI가 해당 이미지를 꺼내 보여줄 수 있음
