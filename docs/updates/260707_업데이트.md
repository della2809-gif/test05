## 자동견적 OCR 플로우 — 건강체크표 사진 → 3단 견적 카드 (신규, B-1)

**왜 만들었나**
자동견적의 핵심인 "건강체크표 사진 → 자동 분석 → 견적 카드"가 미구현 상태였고, 클라이언트가 최종 기준안(`0707 자동견적로직_매니저님전달최종.txt`)을 확정해 전달함.

**뭘 할 수 있나**
1. **사진 → 카드** — 건강체크 그래프 사진 업로드 시 A~J 축 점수 판독 → 양호/보통/경계/불량 판정(불량→경계 우선) → 건강 분석 카드 → 프리미엄/스탠다드/베이직 견적 카드 순 출력.
2. **결정적 계산** — 기준안의 등급별 구성표·축별 추천 제품표·리셋 구성을 그대로 데이터화(`health-quote-rules.ts`)해 규칙 기반 계산. 같은 입력 = 같은 결과. 기본구성 중복 제품은 축 추가분에서 제거.
3. **카드 내용** — 구성·수량·금액·정가 합계·포인트·오토십 10% 할인(쉐이커/지퍼백/셀라비브 화장품은 할인 제외)·최종 금액·예상 첫 수당("예상" 표기 강제)·뉴트리밀 유형/맛 선택·카카오 복사·상담기록 저장. 단가 미확인 구성품은 정직 표기, 적립금은 수치 대신 "별도 안내" 문구(지어내지 않기 원칙).
4. **[확인 필요]** 점수→등급 폴백 구간(양호≤2/보통≤4/경계≤6/불량≥7)은 그래프 원본 기준 미확정 — 상수 분리해둠, 클라이언트 확답 대기(회신문에 질문 포함).

## 수당계산 결정적 계산 — "할 때마다 답이 다름" 수정 (B-2)

**왜 만들었나**
수당계산이 이미지에서 첫 숫자 하나만 뽑아 총 CVP 계산식에 넣던 구조라, 같은 캡처도 매번 다른 결과가 나왔음.

**뭘 할 수 있나**
1. **구조화 판독** — 캡처에서 BC별 {좌, 우} 점수를 구조화 추출(vision json_object)해 기존 `calcCommission`(min(좌,우)×20% 합산)에 연결. 예시(좌406/우3063+5000/0+0/5000 → $81.2) 테스트 재현, 동일 입력 3회 동일 결과 결정성 테스트 포함.
2. **지어내기 금지** — 추출 실패 시 고정 문구로 좌/우 점수를 텍스트로 되물음.
3. **[확인 필요]** 직급별 인컴 기준표($50~$8,000, 퓨처 직급 4주 연속)가 현행 CVP 직급 테이블과 다른 체계인지 정책 확인 필요 — 의도된 기존 로직이라 임의 변경하지 않음.
