## 지플릿 AI 지시사항 미반영 버그 수정

**왜 만들었나**
관리자 화면에서 지플릿별로 AI 지시사항(시스템 프롬프트)을 상세히 작성해 저장해도, 회원 등록·자동 견적처럼 특수 기능이 있는 지플릿에서는 그 내용이 AI에 전달되지 않는 구조적 문제가 있었음. 고객 테스트 중 "회원 등록 지플릿에 프롬프트를 입력했는데 반영이 안 된다"는 피드백을 통해 확인됨. 앞으로 새 지플릿을 추가할 때도 동일한 문제가 반복될 수 있는 상황이었음.

**뭘 할 수 있나**
1. **AI 지시사항 항상 반영** — 어떤 지플릿이든 관리자 화면에서 저장한 AI 지시사항이 빠짐없이 적용됨
2. **신규 지플릿 자동 적용** — 앞으로 새 지플릿을 추가할 때 별도 코드 수정 없이 프롬프트만 입력하면 즉시 반영됨

## DB 입력 마지막 글자 누락 버그 수정

**왜 만들었나**
제품명, 강의 제목 등 DB에 내용을 입력할 때 마지막 글자가 저장되지 않는 현상이 간헐적으로 반복됨. "될 때 있고 안 될 때 있다"는 고객 피드백으로 확인됨. 한글 입력 특성상 마지막 글자가 완성되는 시점에 저장 이벤트가 두 번 실행되며 이전 상태로 되돌아가는 문제였음.

**뭘 할 수 있나**
1. **한글 마지막 글자 정상 저장** — DB 어느 화면에서든 한글 입력 시 마지막 글자가 빠지지 않고 저장됨
2. **전체 입력창 일괄 적용** — 관리자 화면 14개, 사용자 화면 14개 등 서비스 전체 입력창에 동시 적용됨

## 케이스·노출 순서 기능 검증 완료

**왜 만들었나**
이전 세션에서 구현한 케이스 노출 기능과 노출 순서 조정 기능에 대해 실제 환경 검증이 필요했음.

**검증 결과**
1. **케이스 유저 화면 표시** — 케이스 추가 시 유저 채팅 화면에 정상 표시됨 (4개 케이스 전체 확인)
2. **노출 순서 즉시 반영** — 관리자에서 순서 저장 후 채팅 화면으로 이동하면 즉시 반영됨

**주의사항**
채팅 페이지가 이미 열려있는 상태에서 순서를 변경한 경우 새로고침 없이는 바뀌지 않음. 관리자에서 저장 → 채팅 페이지로 이동하는 순서로 사용하면 정상 반영됨.
