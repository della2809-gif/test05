## 대화 목록 사용자 분리

**왜 만들었나**
채팅 사이드바의 대화 목록이 로그인한 사용자 본인 것만 보여야 하는데, 모든 사용자의 대화가 섞여 표시되는 버그가 있었음. 관리자 포함 누구나 다른 사람의 대화 제목을 볼 수 있는 상태였음. 개인정보 보호 및 서비스 기본 요건을 충족하기 위해 즉시 수정이 필요했음.

**뭘 할 수 있나**
1. **대화 목록 분리** — 로그인한 계정 본인이 만든 대화만 사이드바에 표시됨
2. **DB 레벨 격리** — 화면 코드 외에 데이터베이스 접근 정책(RLS)도 함께 적용해, 코드 우회 시에도 타인의 데이터에 접근 불가

## 카테고리 기능 개선

**왜 만들었나**
이전 작업에서 카테고리 추가 UI가 의도치 않게 제거되어, 어드민에서 카테고리를 새로 만들 수 없는 상태였음. 또한 카테고리 관리에서 추가한 카테고리가 항목 추가/수정 화면의 드롭다운에 반영되지 않아 별도로 입력해야 하는 불편함이 있었음.

**뭘 할 수 있나**
1. **카테고리 신규 추가** — "카테고리 관리" 버튼 클릭 후 이름 입력 → "추가" 버튼으로 즉시 생성됨. 생성된 카테고리는 이름 수정·삭제 가능
2. **항목 추가 모달 연동** — 카테고리 관리에서 추가한 카테고리가 항목 추가·수정 화면의 카테고리 드롭다운에도 바로 표시됨
3. **적용 범위** — 스토리, 제품, FAQ, 이미지, 유튜브·강의, 링크, 레퍼런스 파일 등 모든 DB 관리 페이지에 동일하게 적용됨

## 어드민 모달 안정성 개선

**왜 만들었나**
어드민에서 항목을 추가·수정하다가 모달 바깥 영역을 실수로 클릭하거나 ESC 키를 누르면 창이 닫히면서 입력한 내용이 모두 초기화되는 문제가 반복 발생했음. 데이터 입력 작업 자체가 어려운 수준이었음.

**뭘 할 수 있나**
1. **바깥 클릭 무시** — 모달 창 바깥을 클릭해도 창이 닫히지 않음
2. **ESC 키 무시** — 실수로 ESC를 눌러도 입력 내용이 유지됨
3. **X 버튼으로만 닫힘** — 우측 상단 X 버튼을 눌러야만 모달이 닫힘. 모든 어드민 모달에 동일하게 적용됨

## 제품 수정 버튼 수정

**왜 만들었나**
어드민 제품 관리 페이지에서 이미 등록된 제품의 "수정" 버튼을 클릭해도 아무 반응이 없었음. 수정이 필요할 때마다 삭제 후 다시 등록해야 했으며, 이 과정에서 데이터 손실 위험이 있었음.

**뭘 할 수 있나**
1. **수정 버튼 정상 작동** — 제품 항목의 수정 버튼 클릭 시 수정 모달이 정상적으로 열림

## PDF 업로드 수정

**왜 만들었나**
레퍼런스 파일 섹션에 PDF를 업로드하려 했으나 업로드가 안 되는 오류가 있었음. 이희정님이 영양제 관련 PDF 자료를 업로드해 제품 추천에 활용할 계획이었으나 진행이 막힌 상태였음.

**뭘 할 수 있나**
1. **PDF 업로드 가능** — 레퍼런스 파일에 PDF 파일 정상 업로드됨. 업로드한 내용은 자동으로 분석되어 AI 채팅에서 검색 가능

---

## YouTube·이미지 DB 검색 수정

**왜 만들었나**
YouTube DB에 자료를 저장해도 채팅에서 검색되지 않는 문제가 있었음. 이미지 DB의 경우 설명 문구(스크립트)가 없는 항목은 채팅에서 불러오지 못했음. 두 경우 모두 저장 시 AI 검색용 색인이 생성되지 않아서 발생한 문제였음.

**뭘 할 수 있나**
1. **YouTube 자료 검색** — YouTube DB에 저장한 자료가 채팅에서 정상 검색됨. 기존에 저장된 항목은 한 번씩 다시 저장해야 검색 색인이 생성됨
2. **이미지 검색 일관성** — 설명 문구 없이 제목만 있는 이미지도 채팅에서 불러올 수 있음

## 회원 관리 UX 개선

**왜 만들었나**
회원 등록 완료 후 목록 페이지에 등록된 회원이 바로 보이지 않았음. 등록이 성공했는지 확인할 방법도 없어 혼란스러운 상황이었음. 메인 화면에 "회원 관리"와 "회원 등록" 두 항목이 별도로 표시되어 구조가 중복된 문제도 있었음.

**뭘 할 수 있나**
1. **등록 후 즉시 목록 이동** — 회원 등록 완료 시 자동으로 회원 목록 화면으로 이동됨
2. **등록 완료 메시지** — 등록이 완료되면 확인 메시지가 표시됨
3. **메뉴 명칭 통일** — 메인 화면에서 "회원 관리" 항목 하나로 통일됨

## 텍스트 입력란 높이 조절

**왜 만들었나**
스크립트, 요약, FAQ 답변 등 긴 내용을 입력하는 칸이 너무 짧아 입력 중 전체 내용을 확인하기 어려웠음. 특히 유튜브 강의 스크립트처럼 긴 텍스트를 붙여넣을 때 불편함이 컸음.

**뭘 할 수 있나**
1. **높이 자유 조절** — 입력란 하단을 마우스로 드래그해 원하는 높이로 늘릴 수 있음. 모든 긴 텍스트 입력 칸에 적용됨

## 이미지 여러 장 업로드

**왜 만들었나**
이미지 DB에 항목을 등록할 때 한 번에 1장씩만 업로드 가능했음. 제품 사진처럼 여러 장이 한 세트인 경우 반복 작업이 필요했음.

**뭘 할 수 있나**
1. **다중 업로드** — 이미지 DB에서 파일 선택 시 여러 장을 동시에 선택해 한 번에 업로드 가능함. 각 이미지는 번호가 붙어 개별 항목으로 등록됨
