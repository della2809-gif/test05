# GENIEA 폴더 정리 진단

작성일: 2026-07-11

## 현재 판단

이 저장소는 Next.js 앱 루트에 앱 코드, 운영 문서, DB 반영 산출물, 에이전트 캐시가 섞여 있다. 앱 자체의 핵심 구조는 `src`, `supabase`, `scripts`, `e2e`, `public`, 설정 파일 중심으로 유지하면 되고, 난립 문제는 로컬 자료와 생성 산출물을 별도 영역으로 분리하는 방식이 적합하다.

## 루트에 남길 것

- `src/`, `public/`, `supabase/`, `scripts/`, `e2e/`
- `package.json`, `package-lock.json`, `next.config.ts`, `tsconfig.json`, `vitest.config.ts`, `playwright*.config.ts`, `eslint.config.mjs`, `postcss.config.mjs`
- `README.md`, `AGENTS.md`, `CLAUDE.md`, `PROMPT_SUITE.md`
- `.env*`, `.vercelignore`, `.gitignore`

## 문서/자료 영역

- `00_decisions/`: 구조적 판단과 정책 결정 기록. 기존 스킬/문서 규칙과 맞물려 있으므로 유지.
- `00_requirements/`: 클라이언트 요구사항, 원본 자료, DB 입력안 등 로컬 기획 자료.
- `01_specs/`: 운영 매뉴얼, 페이지/기능/디자인 스펙.
- `02_todo/`: 수정요청, 캡처, 작업목록.
- `03_화상회의/`: 회의록 원본.
- `docs/`: 운영 문서의 단일 루트. 업데이트, 운영 기록, 인수인계, 프롬프트, 교육자료, 아카이브를 하위 폴더로 분리한다.

## 산출물/캐시 영역

- `.next/`, `node_modules/`, `test-results/`: 재생성 가능한 개발 산출물.
- `.hermes/`, `.tmp-knowledge-mcp/`, `.gstack/`, `.omc/`: 에이전트/도구 로컬 상태.
- `geniea_outputs/`, `geniea_db_batches/`, `geniea_thread_memory/`, `knowledge-wiki/`, `geniea_workshop_review/`: DB 작업 산출물과 임시 작업 메모. 앱 런타임 소스가 아니므로 git 상태에서는 기본적으로 숨기는 쪽이 맞다.

## 1차 정리 내역

- 루트 DB 입력안 파일 5개를 `00_requirements/07_비정형_데이터/DB입력안/`으로 이동.
- 루트 `tmp_ocr.swift`를 `scripts/tmp_ocr.swift`로 이동.
- `scripts/ocr_reference_pdfs_to_txt_and_upload.mjs`가 이동된 Swift OCR 헬퍼를 절대 경로로 호출하도록 수정.
- `.gitignore`에 로컬 에이전트 상태와 GENIEA 작업 산출물 폴더를 추가.

## 2차 정리 제안

- `04_docs/260*_업데이트.md`와 `docs/260*_업데이트.md`를 `docs/updates/`로 통합 완료.
- `04_docs/채팅_자동진입_설명.md`처럼 운영 설명에 가까운 문서는 `docs/operations/`로 이동 완료.
- `geniea_outputs/` 내부 파이썬/엑셀/백업/이미지 산출물은 유지하되, 필요하면 내부만 `db/`, `client-deliverables/`, `screenshots/`, `backups/`로 재분류.
- `.next/`, `.tmp-knowledge-mcp/`는 용량이 크므로 필요 시 삭제해서 디스크를 회수할 수 있다. 삭제 후에는 빌드/도구 실행 시 다시 생성된다.
