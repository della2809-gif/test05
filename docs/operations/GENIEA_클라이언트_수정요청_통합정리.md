## 5. 이미 확인된 운영/안전 기준

- 클라이언트 요청은 실서버 기준으로 받되, 봇/작업자는 preview/test 서버를 클라이언트에게 안내하지 않음.
- 명확한 버그/문구/스타일/안전한 테스트 편의 요청은 로컬 수정 준비 가능.
- GitHub push, main 반영, production deploy는 사용자/매니저 승인 없이 진행하지 않음.
- DB/검색/계산/추천/사용자 화면에 영향이 있는 변경은 먼저 검토표 또는 샘플 입력안을 작성한다. 검토표 작성 단계는 승인 없이 진행 가능하며, 실제 DB 저장/대량 반영 직전에만 사용자/매니저 승인을 받는다.
- 실전미팅/스크립트 DB는 원본 보관용과 DB 반영용 익명화/정리본을 분리.
- 비포애프터 이미지는 본인 동의, 가족 보유+동의 확인, 공식 강의/유튜브 자발 공개 자료인 경우 사용 가능 기준으로 승인 확인됨.
- Slack 파일 첨부는 실제 파일 카드/다운로드 버튼이 확인되기 전까지 “첨부 완료”라고 말하지 않는 것이 안전함.
- 수당 계산 운영정책은 `GENIEA_운영정책.md` 3-10에 확정 반영됨. 핵심 기준은 소실적 125점 이상, 기본 수당 20%, 세일즈 보너스 10%/6개월, 3BC 공유·누적 구조, 공식 보상플랜 우선 안내임.
- 여행 달성 계산 운영정책은 `GENIEA_운영정책.md` 3-11에 확정 반영됨. 핵심 기준은 프로모션 조건표 + 유사나 문자 현황 + 현재 수당 구조 + 승급/팀 성장 흐름을 함께 보는 시뮬레이션이며, 이미 도전 중인 사람과 처음 도전하는 사람을 분기함.
