## 3. 남은 작업과 "왜 코드로 못 묶는지" (오늘 확인된 사실)

### CR-038 대용량 PDF (뉴트리셔널 써플리먼츠) — 코드 아님, 데이터 작업
- 4.5MB 제한은 **Vercel 플랫폼 하드 제한**(서버리스 함수 본문 최대). 업로드가 함수를 통과하므로 107MB는 구조상 불가. 숫자만 올린다고 안 됨.
- 게다가 책이 **스캔본**이라 외부 OCR 없이는 텍스트가 안 나옴.
- **현실적 해결**: 외부 OCR → 소용량 텍스트(.txt/.md)로 추출 → 업로드(4.5MB 이하) → 기존 파이프라인이 자동 청크·임베딩. **김나연님이 OCR 텍스트를 제공해야 진행 가능.**
- "스토리지 직접 업로드" 코드 기능(B/C)은 스캔본엔 무용하고 활용 빈도 낮아 현재는 과투자로 판단 → 보류.
- 관련: CR-023, CR-043(Product Rating 구조화). 표 캡처/OCR 원본 확보가 선행.

### CR-010 여행 계산 — 데이터 의존
- 운영정책(3-11)은 반영됨. 실제 기능 구현은 **프로모션 조건표(Image DB) + 유사나 문자 캡처/OCR 샘플 + 도전중/처음도전 분기** 데이터가 있어야 정확히 만듦. **김나연님 데이터 제공 필요.** 별도 트랙으로 진행 권장(다른 작업을 묶어 막지 말 것).

### P0 검토표/입력안 (코드 아님, 문서 작업)
- Script DB 전체 재구조화(CR-017/029/036), 실전미팅 DB 질문형 코칭(CR-036), Image DB 업로드 가이드(언주님), 자유채팅 추가 케이스 등은 "검토표/입력안" 산출물 작업. 실제 DB 반영 전 승인 필요.
