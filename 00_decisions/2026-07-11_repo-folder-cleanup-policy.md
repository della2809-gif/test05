# 저장소 폴더 정리 정책

## [2026-07-11] 앱 루트는 실행 소스 중심으로 유지하고, 로컬 자료와 생성 산출물은 분리한다

**결정**: GENIEA 저장소 루트에는 앱 실행과 개발에 직접 필요한 소스/설정만 남기고, 요구사항·DB 입력안·작업 산출물·에이전트 캐시는 문서/로컬자료/생성산출물 영역으로 분리한다.

**이유**: 현재 루트에는 Next.js 앱 코드와 클라이언트 자료, DB 반영 산출물, 도구 캐시가 함께 놓여 있어 탐색 비용과 git 상태 노이즈가 크다. `src`, `supabase`, `scripts`, `e2e`, `public` 중심의 앱 구조는 유지하고, 재생성 가능하거나 작업 산출물 성격인 폴더는 `.gitignore`로 숨겨 루트와 변경 목록을 안정화한다.

**버린 대안**:
- 모든 폴더를 `docs/` 아래로 통합 — `00_decisions`, `00_requirements`, `01_specs`, `02_todo`의 기존 작업 흐름과 스킬 규칙을 깨뜨릴 수 있어 보류.
- `geniea_outputs`, `geniea_db_batches`를 삭제 — DB 작업 증적과 전달 산출물이 섞여 있어 손실 위험이 있으므로 삭제하지 않고 ignore 처리.
- `.next`, `node_modules`까지 이동 — Next.js/npm 기본 동작을 벗어나 개발환경을 불안정하게 만들 수 있어 유지.

**영향 범위**: 루트 파일 배치, `.gitignore`, 로컬 DB 입력안 위치, OCR 보조 스크립트 위치.


## [2026-07-11] 2차 문서 통합

**결정**: `04_docs/`와 `docs/`에 나뉘어 있던 운영 문서를 `docs/` 단일 루트로 합치고, 문서 성격별로 `updates/`, `operations/`, `handovers/`, `prompts/`, `training/`, `archive/`에 배치한다.

**이유**: 날짜별 업데이트 문서와 운영 문서가 두 루트에 중복되어 있어 새 작업자가 문서 위치를 예측하기 어렵다. 앱 소스와 무관한 문서는 `docs/` 아래에서 역할별로 분리하는 것이 탐색 비용이 가장 낮다.

**버린 대안**:
- `04_docs/` 유지 — 기존 중복 구조를 남겨 앞으로도 업데이트 문서가 양쪽에 생성될 가능성이 크다.
- 모든 문서를 `docs/operations/`에 몰아넣기 — 업데이트, 인수인계, 교육자료의 성격이 섞여 다시 평면화된다.

**영향 범위**: `04_docs/` 문서, `docs/` 루트 문서, 문서 내부 경로 참조.
