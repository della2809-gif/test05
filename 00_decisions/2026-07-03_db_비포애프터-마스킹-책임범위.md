# 의사결정: 비포애프터 이미지 — 얼굴 없는 사진만 우리가 반영, 마스킹·민감사례는 클라이언트 처리

## 배경
- 클라이언트가 6/28~30 전달한 비포애프터 체험사례 이미지(~170장)가 미반영 상태였음
- 6/28 봇(지디)이 "눈 마스킹 가능합니다, 묶음 지정해주시면 처리하겠습니다"라고 답한 기록이 있음
- 그러나 이미지 가공(마스킹)은 계약 업무 범위에 없음

## 결정 (나연 매니저, 2026-07-03)
- (a) 얼굴이 안 나오는 사진(등/피부 부위, 인바디 캡처 등): **우리가 반영**
- (b) 얼굴 노출 사진의 눈 마스킹, (c) 민감 질환(암 등) 사례의 표현 수위 처리: **클라이언트가 직접 처리 후 관리자 페이지 > 이미지 DB에서 직접 업로드** (DB 대량등록 대행도 계약 범위 외 — 어드민 업로드 시 검색 반영은 자동)

### (a) 반영 방식 결정 (실행 중 확정)
- **묶음(사례) 단위로 이미지들을 한 장의 비포애프터 카드(가로/그리드 몽타주)로 합쳐 admin_images 1행씩 반영.** (route.ts는 1파일=1행 구조인데, 전/후를 한 화면에 보여야 비포애프터 의미가 살고, 같은 사례가 여러 행으로 중복 노출되는 것을 막기 위함.) → 대안이던 "장별 개별 행(40행)"은 채택 안 함.
- **결과: (a) 10묶음(40장) → 카드 10건 반영, admin_images 93→103, document_chunks 88→98.** 원본은 raw/, 카드는 cards/, 롤백 로그는 upload_log_admin_images.json에 보존.
- **봇 판정 재검증:** 콘택트시트 육안 확인에서 봇이 "마스킹 불필요"로 본 b077(여성 3인 얼굴 홍보카드)·b106(다인 미팅 사진)이 실제로는 얼굴 노출이라 (a)→(b)로 이동. 봇 메타만 믿지 말고 (a) 반영 전 육안 확인 필수.

## 이유
- 마스킹·의료정보 가공은 계약 범위 외 작업
- 개인정보/의료정보의 공개 기준은 원저작자(클라이언트)가 정하는 것이 법적으로도 안전
- 단, 봇이 과거에 마스킹을 약속한 기록이 있으므로 회신은 "계약 외"가 아니라 "개인정보라 대표님이 기준을 정해 처리하신 원본을 주시면 바로 반영" 프레임으로 (신뢰 이슈 방지)

## 검증 기준
- (a)군 반영 후 admin_images 건수가 (a) 묶음 수(카드 수)만큼 증가 (장수가 아니라 묶음 단위) — 실측 93→103(+10)
- (b)/(c)군은 DB에 없음 (클라이언트 원본 수령 전까지)
- 신규 카드가 관련 질의에서 검색(match_documents_by_source)에 노출됨 — 스팟 체크 통과
- 클라이언트 안내문은 계약 언급 없이 "대표님이 처리 후 관리자 페이지에서 직접 업로드 → 검색 자동 반영" 프레임 사용
