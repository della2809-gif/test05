# 의사결정: CR-034 레퍼런스 DB(admin_files) 활성/비활성 토글

작성일: 2026-06-27

## 배경
- 레퍼런스 DB에 실전미팅 원본·책 PDF 등 운영자용 자료를 보관하되, 필요 시 RAG 검색/답변에서 제외할 수 있어야 함(CR-034). 3개 DB 추가작업(이미지DB, 책 PDF, 실전미팅 DB화)의 선행 작업.
- 기존 `admin_files`에는 활성화 필드가 없었고, 레퍼런스 검색은 RAG 전용(`document_chunks`, `source_id`로 원본 연결)이라 단순 버튼 추가로 안 되고 DB 컬럼 + 검색 + 화면을 함께 맞춰야 함.

## 선택지 (RAG 비활성 제외 방식)
- A. 앱단 사후필터: `searchDocumentsBySource`가 `source_id`를 반환하지 않아 매핑 불가 → 탈락
- B. 전용 RPC에서 `admin_files` JOIN + `is_active` 필터 (기존 `match_youtube_documents` 패턴)
- C. 토글 시 청크 삭제/재적재 (무겁고 비가역적)

## 결정
- **B 채택.** 김나연님이 다음 작업으로 CR-034 지정(2026-06-27).
- 구현(로컬):
  - 마이그레이션 A `20260627000000_admin_files_add_is_active.sql`: `admin_files.is_active BOOLEAN NOT NULL DEFAULT true`. 기존 파일 전부 활성 → 적용해도 동작 변화 0.
  - 마이그레이션 B `20260627000001_match_reference_documents.sql`: `match_reference_documents` RPC. `document_chunks JOIN admin_files ON af.id=dc.source_id`, `source_type='admin_files' AND af.is_active=true` 필터. 반환 컬럼은 `match_documents_by_source`와 동일.
  - `src/lib/rag.ts`: `searchReferenceDocuments()` 추가. 1순위 새 RPC, **graceful 폴백**(RPC 없으면 `searchDocumentsBySource(...,"admin_files")`로 위임 → 마이그레이션 전에도 검색 안 깨짐, is_active만 미적용). 활성검색이 빈 결과여도 폴백으로 새지 않음(비활성 재포함 방지).
  - `src/lib/openai.ts:437`: 레퍼런스 로더를 `searchReferenceDocuments`로 교체.
  - `src/app/api/admin/files/[id]/route.ts`: **PATCH 신규**(DELETE와 동일 인증: getUser→profiles.role='admin'). `{is_active}` 업데이트.
  - `src/app/admin/files/page.tsx`: 활성/비활성 토글(Power/PowerOff) + 뱃지 + 비활성 행 opacity-50. 목록·스토리지엔 유지.
  - `src/types/database.ts`: `AdminFile.is_active` 추가.

## 이유
- B는 FAQ 등 다른 source_type 검색에 영향 0(전용 RPC 분리). 앱단 필터는 source_id 부재로 불가. 청크 삭제(C)는 비가역적이라 토글에 부적합.
- graceful 폴백으로 마이그레이션 적용 순서/타이밍과 무관하게 레퍼런스 검색이 항상 동작.

## 영향 범위
- 데이터 구조: `admin_files`에 `is_active` 컬럼 추가, 신규 RPC `match_reference_documents`.
- 코드: rag.ts, openai.ts(레퍼런스 로더), files [id] route(PATCH), files 관리자 페이지, database 타입.

## 검증 기준
- 관리자 레퍼런스 화면에서 파일을 비활성하면 목록엔 남고(흐림+비활성 뱃지), 챗봇 답변에서 그 파일이 더 이상 검색되지 않는다.
- 활성 상태 파일만 `match_reference_documents`로 검색된다. 활성 매칭이 0건이어도 비활성 파일이 재포함되지 않는다.
- 마이그레이션 적용 전: 레퍼런스 검색은 폴백으로 정상 동작(필터만 미적용), UI 토글은 컬럼 부재로 실패할 수 있음.
- tsc 통과, 신규 lint 패턴 0(supabase 캐스트는 기존 패턴).

## 되돌리기
- 코드: 단일 기능 커밋 revert. RPC/컬럼은 DROP으로 원복 가능.
- 적용 순서: 마이그레이션 A→B 적용 후 배포. 미적용 시 폴백으로 안전.

## 상태
- 로컬 구현·정적검증 완료(executor 구현 + 별도 리뷰 패스). **live DB 마이그레이션 적용·push·배포는 김나연님/매니저 승인 후.** 로그인 게이트라 토글 런타임 검증은 마이그레이션 적용된 환경에서 수동 확인 필요.
