# 의사결정: schedules.life_layer "유산화" → "유사나" 오타 정정 (2단계 확장→축소 이관)

## 배경
- "유산화"는 음성 받아쓰기 오류로 굳어진 오타. 실제 값은 "유사나".
- 일정 라이프레이어 분류값으로 코드 5곳 + DB CHECK 제약 + 기존 28행에 하드코딩.
- 조사: `docs/operations/GENIEA_W1_조사결과_2026-07-02.md` W1-A §2. 나연 매니저 승인 완료.

## 선택지
- (A) 라이브 CHECK에서 즉시 '유산화'를 제거하고 '유사나'만 허용 + 코드 치환.
- (B) 라이브 CHECK를 '유산화'+'유사나' 둘 다 허용으로 **확장**만 지금 하고, '유산화' 제거는 신규 코드 배포 시점의 마이그레이션으로 미룸.

## 결정
- **(B) 채택.** 지금은 확장 + 기존 행 UPDATE + 코드 치환까지만 라이브 반영. '유산화' 제거는 배포용 마이그레이션 파일로 준비(미적용).

## 이유
- 현재 프로덕션에는 **구 코드가 배포된 상태**라 회원 등록마다 `life_layer:"유산화"` INSERT가 계속 발생(`api/contacts/route.ts:149,179`, `[id]/route.ts:189`).
- 코드는 커밋/배포 없이 파일 수정만 한 상태 → 신규 코드가 라이브가 아님.
- 이 시점에 CHECK에서 '유산화'를 제거하면 구 코드의 INSERT가 제약 위반으로 **즉시 실패**(일정 저장 깨짐). 확장-후-배포시점축소가 유일한 무중단 순서.
