# 의사결정: 고객 성향(논리적/감성적/실용적)을 고객 단위로 저장하고, 모르면 되묻아 고객 발송 문구 톤에 반영

## 배경
- 이 AI는 고객이 쓰는 챗봇이 아니라 직원(상담자)이 쓰는 보조 도구다. 그러므로 성향의 주체는 직원이 아니라 **고객**이어야 한다.
- 어제(2026-07-02) 배포된 "사용자(직원) 성향" 기능(커밋 3c1e2f8, 설정 페이지 라디오 + profile.personality 주입)은 성향 주체를 직원으로 잡은 **개념 오류**로 판정됐다.
- 요구 흐름: 고객 안내가 필요한 상황 → 고객 성향을 모르면 AI가 "이 고객은 논리적/감성적/실용적 중 어느 쪽인가요?"를 역으로 질문 → 직원이 답하면 → 고객 발송 문구가 그 성향 톤으로 출력. 이미 아는 고객(회원 DB에 저장)은 되묻지 않고 바로 적용.

## 선택지
- 모를 때 동작: **(채택) 기본톤 1차 출력 + 되묻기 병행** vs 성향 확인 전 출력 보류.
  - 채택 이유: 이 도구는 직원이 상담 중 바로 복붙하는 속도용이다. "무조건 되묻고 답 오기 전까지 보류"는 상담 속도를 깎는다. 클라이언트가 "항상 물어보고 나서만 출력"을 원하면 규칙 한 줄로 전환 가능.
- 저장 원천: **(채택) contacts.personality (고객 단위)** vs conversations 단위 vs profile(직원) 단위.
  - 채택 이유: 직원 1명이 고객 수십 명을 상담하므로 성향은 고객(회원)에 붙어야 확장성이 있다.

## 결정
- **P1** — 프롬프트 규칙 전환 + 설정 라디오 제거:
  - 설정 페이지 "출력 성향" 라디오 제거(로그아웃/탈퇴 유지).
  - `src/lib/openai.ts`: USER_PERSONALITY_* → CUSTOMER_PERSONALITY_*(대상=고객 발송 문구)로 개념 전환. 항상 주입되는 `CUSTOMER_PERSONALITY_PROMPT` 규칙 블록((a)대화 언급 → (b)회원 저장 성향 → (c)둘 다 없으면 기본톤+되묻기) 추가. 기존 PERSONALITY_PROMPT 및 직원용 personality 파라미터 전량 제거.
  - messages/route.ts·reorganize/route.ts의 profiles.personality 조회/전달 제거.
- **P2** — contacts.personality + 회원 연동:
  - `contacts.personality text NULL CHECK (logical/emotional/practical)` 컬럼 추가(라이브 적용 완료 + 마이그레이션 파일).
  - 회원 등록/수정 폼에 성향 선택(선택 안 함 허용), 회원 상세에 표시(미설정=미파악).
  - 회원등록 지플릿 `register_contact` tool에 personality optional 인자 추가 + 등록 처리에서 저장.
  - 회원 매칭 자동 주입(계층 C): openai.ts에 회원 로더 추가 — 사용자 메시지에 저장 회원 이름이 등장하면 그 회원 성향을 "성향: 감성적" 형태로 컨텍스트에 주입.

## 이유
- 성향의 주체를 고객으로 바로잡아야 실제 상담 흐름(직원이 고객별로 다른 톤의 문구를 뽑는 것)과 맞는다.
- 저장(contacts) + 되묻기(프롬프트)를 함께 가야, 아는 고객은 재질문 없이 즉시 적용되고 모르는 고객만 되묻어 반복 질문 피로를 없앤다. (P1만 하면 저장이 없어 매번 되물어 오히려 불편)

## 영향 범위
- `src/lib/openai.ts` — 파라미터/상수/규칙 블록/회원 로더/register_contact tool schema
- `src/app/api/conversations/[id]/messages/route.ts` — profiles.personality 제거, 등록 시 personality 저장
- `src/app/api/archives/[id]/reorganize/route.ts` — personality 제거(mode 유지)
- `src/app/(main)/settings/page.tsx` — 출력 성향 섹션 제거
- `src/app/(main)/contacts/new/page.tsx`, `contacts/[id]/edit/page.tsx`, `contacts/[id]/page.tsx` — 성향 선택/표시
- `src/lib/__tests__/openai-personality.test.ts` — 고객성향 개념으로 재작성
- `supabase/migrations/20260703000000_add_contact_personality.sql` — 컬럼 DDL (라이브 반영 완료)

## 검증 기준 (설계 문서 §6 인용)
- 성향 미저장 고객 상담 → 발송 문구 + "고객 성향이 어느 쪽인가요?" 질문이 함께 출력된다.
- "감성적이야"라고 답하면 → 발송 문구가 공감 톤으로 재출력된다.
- 성향이 저장된 회원 이름으로 상담 → 되묻지 않고 바로 해당 톤으로 출력된다.
- 직원용 섹션(상담 판단/추천 방향)의 톤은 성향과 무관하게 동일하다.

## 되돌리기
- 쉬움. 프롬프트 규칙 블록·회원 로더·성향 UI/컬럼만 제거하면 원복. contacts.personality 컬럼은 NULL 허용이라 잔존해도 무해.
