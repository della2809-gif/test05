  CR-004(유튜브 추천 부정확), CR-007(제품정보검색 지플릿 품질 저하), CR-008(전체 DB 검색 품질)에 대해

## 규명된 근본 원인 (검증 기준의 근거)
- **CR-007 (회귀 확정)**: `src/lib/openai.ts:143-169`
  - 제품 로더에 `hasKeyword(userLastMessage, PRODUCT_KW)` 게이트 추가(`:145`) → "헬스팩 알려줘"처럼
    PRODUCT_KW(제품/영양제/비타민/오메가/유사나/복용/성분/효능/가격/구매/건강기능/추천)가 없는 질의는
    제품 DB가 통째로 미주입 → 가격·유사IQ 상세링크 누락.
  - 이전 프롬프트의 `"반드시 아래 원문 그대로 출력하세요. 임의로 요약·재구성 금지"` 지시가 삭제됨 → 형식 붕괴.
  - 이미지 로더(`:192`)는 게이트 없이 매 메시지 실행 + 임계값 0.25(느슨) → 엉뚱한 이미지.
  - 링크 로더(`:183`)는 `LINK_KW` 게이트 → "헬스팩 알려줘"엔 키워드 없어 링크 미주입.
- **CR-004**: `openai.ts:223-252` + `rag.ts` + `supabase/migrations/20260604000001_match_youtube_documents.sql` + `src/app/admin/youtube/page.tsx:195`
  - `match_youtube_documents`가 `document_chunks`에서 **청크 단위 top-5**만 코사인으로 반환 → 영상 단위 보장 없음
    (클라이언트 지적 "Top 5 chunk 기준, 영상 5개 보장 아님"과 일치).
  - 제목 정확매칭/필드 검색 경로 부재. 제목은 임베딩 첫 800자 청크에만 들어가 자막 청크들과 동등 경쟁 → "제목 바꿔도 흔들림".
  - 임계값 0.25 + 프롬프트가 부분 결과를 `"DB에 존재하는 전부입니다 … 이 외엔 존재하지 않습니다"`로 단언 → 실제 영상을 "없다"고 답하게 만듦.
- **CR-008**: `openai.ts`
  - ilike 검색 대상 필드가 좁음 — templates는 `title`만(`:86`), blocks는 `title+category`(`:110`),
    products는 `name+keywords+symptoms`(`:148`). `content/description/category(템플릿)/tags/aliases`는 미검색 → "큰 카테고리/본문 기준" 데이터 누락.
  - `extractSearchTerms`(`:26`)가 불용어("알려줘" 등)도 검색어로 사용 → 잡음.
  - 1순위 매칭 실패 시 관련성 없는 "최근 N개"를 주입하고 "이것만 사용"으로 지시 → 엉뚱한 결과.

## 선택지
- **A. 한 건씩 수정 → 로컬 확인 → 승인 → 다음** (순서 CR-007 → CR-004 → CR-008), CR-007 직전에 지플릿 db_sources 확인(C)을 흡수
- **B. 셋 다 수정안을 한 번에 만들어 일괄 리뷰**
- **C. 라이브 DB에서 제품정보검색 지플릿 db_sources부터 확인 후 시작**

## 결정
- **A 채택.** CR-007 → CR-004 → CR-008 순서로 한 건씩 수정·검증한다. CR-007 시작 직전에 C(지플릿 db_sources 확인)를 1회 점검으로 포함한다.

## 이유
- 세 건 모두 검색 로직을 건드리는데, 해당 화면들은 로그인 게이트라 자동 검증이 어렵다. 묶어서 바꾸면(B)
  어느 변경이 정확도를 개선/악화시켰는지 격리가 불가능하다 → systematic-debugging의 "한 번에 한 변수" 원칙을 따른다.
- 순서는 위험·난이도순: CR-007은 명확한 회귀이고 수정이 가장 가볍다(게이트 완화 + 삭제된 프롬프트 한 줄 복구).
  CR-004는 영상 단위 집계 등 RPC까지 손봐야 해 중간. CR-008은 가장 광범위/구조적이라 마지막.
- C를 007에 흡수: 제품정보검색 지플릿의 `db_sources`만 코드가 아닌 라이브 DB(admin_giplets)에 있어 유일한 미확정 가정이며, 관리자 화면 1회 확인으로 해소된다.

## 영향 범위
- `src/lib/openai.ts` (스마트 로딩 로더 전반)
- `src/lib/rag.ts` (임계값/검색 헬퍼)
- `supabase/migrations/*` (CR-004 영상 단위 집계용 RPC 신규/수정 가능성)
- `src/app/admin/youtube/page.tsx` (임베딩 텍스트 구성, 필요 시)
- 런타임 DB `admin_giplets`(제품정보검색 지플릿 db_sources) — 코드 외 확인 대상

## 검증 기준
- CR-007: "헬스팩 알려줘"(PRODUCT_KW 없는 질의)에 제품 정보·가격·유사IQ 상세링크가 출력되고, 형식이 예전처럼 유지된다.
- CR-004: 유튜브 질의 시 **서로 다른 영상**이 의도한 개수만큼 반환되고, 제목을 바꾸면 추천 결과가 반영되며, DB에 있는 영상을 "없다"고 답하지 않는다.
- CR-008: `content/description/category`에만 키워드가 있는 데이터도 검색되며, 매칭 실패 시 무관한 항목을 "이것만 사용"으로 단언하지 않는다.
