# 의사결정: FAQ 호전반응 DB 반영 + FAQ 검색 경로 분리 + match_documents_by_source 마이그레이션 적용

작성일: 2026-06-26

## 배경
- 통합정리 문서의 DB 작업 중 "FAQ 호전반응"을 첫 대상으로 선택.
- 이전 세션이 만들어 둔 입력안(`geniea_outputs/db_upload_batches/..._healing_reaction_faq.csv`, 30행)을 실제 prod DB에 반영하기로 함.
- 사용자 요구: 실제 DB 반영까지 하되, 기존 DB와 충돌 없이 확실히 검토 후 진행.

## 결정
1. **FAQ 데이터 반영**: 호전반응 FAQ 30건을 `faqs` 테이블에 insert + `document_chunks`(source_type=faq)에 임베딩. tags에 `내부용` 마커 추가.
2. **안전문구**: faqs 테이블에 전용 컬럼이 없어, FAQ 지플릿(`admin_giplets.giplet_key=faq`) `system_prompt`에 안전 배너 규칙을 넣는 방식 채택(태그는 마커만).
3. **FAQ 검색 경로 분리**: `rag:faqs`를 generic `searchDocuments`(임계값 0.6, 전체 출처 혼합)에서 떼어내 `searchDocumentsBySource(faq, threshold 0.35)`로 전환.
4. **prod 마이그레이션 적용**: 미적용 상태였던 `20260611000000_match_documents_by_source.sql`을 `supabase db push`로 prod에 적용(미적용 마이그레이션은 이 1건뿐임을 dry-run으로 확인).

## 이유
- 기존 `faqs`는 2행(호전반응 0건)이라 **중복·충돌 없음**을 확인 후 반영.
- 한국어 임베딩 유사도 실측 상위값이 ~0.43이라 임계값 0.6에선 FAQ가 사실상 검색되지 않았고, generic 검색은 비-FAQ 청크(유튜브 987개 등)에 묻혔다. faq 출처 전용 + 0.35가 올바른 경로.
- `searchDocumentsBySource`의 1순위 RPC(`match_documents_by_source`)가 prod에 없어 깨진 fallback(전체 top-N 후 필터)을 타고 있었음 → 이 RPC 적용이 FAQ·레퍼런스(admin_files) 검색의 공통 전제.

## 영향 범위
- prod DB: `faqs`(+30행), `document_chunks`(+30 faq 청크), `admin_giplets`(faq system_prompt), 신규 함수 `match_documents_by_source`.
- 코드: `src/lib/openai.ts`(rag:faqs 전용 검색 블록) — 커밋 b52b414.

## 검증 기준 / 현재 상태
- ✅ faqs 32행(호전반응 30 + 기존 2), 임베딩 30, tags 내부용 30, 배너 적용, RPC 적용.
- ✅ 잘 표현된 질의는 정확 매칭(예: "병원 약 끊어도 되나요" → 해당 FAQ sim 0.533).
- ⚠️ **한계**: 조사·패러프레이즈("변비랑 설사…" vs "변비나 설사는…", "미열이 나요")는 임베딩 유사도가 낮고 ilike는 조사로 빗나가 적중 실패. 적중률이 표현에 따라 들쭉날쭉.
