## 작업 완료 후 보고 규칙

작업이 끝날 때마다 아래 형식으로 **반드시** 보고한다. 요약 수준 금지 — 최대한 상세하게 작성한다.

### 보고 형식

```
## ✅ 작업 완료 보고

### 📁 변경된 파일
- `파일경로` — 무슨 파일인지 한 줄 설명

### 🔧 기술적으로 한 일 (개발자용)
- 어떤 함수/컴포넌트를 수정했는지
- 어떤 로직을 추가/변경했는지
- DB 스키마나 API 엔드포인트 변경 여부

### 🗣️ 쉽게 말하면 (비개발자용)
- 사용자 입장에서 뭐가 달라졌는지
- 어디서 어떻게 확인할 수 있는지
- 예시: "이제 완료 버튼을 누르면 저장이 됩니다. 주문 주기 화면에서 수정 후 확인해보세요."

### ⚠️ 주의사항 / 다음 할 일
- 이 작업으로 인해 테스트해야 할 것
- 연관된 다음 작업
```

### 추가 규칙
- 세션 종료 전 `/update-dev` 스킬로 `docs/YYMMDD_업데이트.md` 파일에 전체 세션 작업 내역을 저장한다.
- 비개발자도 이해할 수 있는 언어로 쓴다. "컴포넌트", "훅", "API" 같은 용어는 괄호 안에 쉬운 설명을 병기한다.
- 02_todo 폴더의 해당 항목을 완료 처리(체크)한다.
