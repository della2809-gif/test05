# [화상] ChatGPT API 연동 기반 AI 서비스 MVP 개발 (핵심 기능 1~2개) - 2026_05_29 - 스크립트

## 요약

5월 29일 화상회의에서는 지니아 MVP의 버그 수정 현황, 유튜브/스크립트 데이터 활용 방식, 지플릿과 케이스 관리의 역할 구분, 메인 화면 노출 순서 및 사용 흐름 개선 방향, 향후 운영/유지보수 방식에 대해 논의했다.

핵심 결론은 다음과 같다.

- 패키지 저장 오류, 레퍼런스 파일 업로드 오류 등 기존 버그는 김나연이 수정해 두었음.
- 유튜브 자막 자동 추출은 편의 기능으로만 보고, 자동으로 불러오지 못하는 경우 별도 자막/스크립트를 붙여 넣는 방식으로 운영하기로 함.
- 유튜브 데이터는 요약만 보여주는 것이 아니라, 사용자가 클릭 가능한 링크 형태로 노출되도록 수정하기로 함.
- 이희정이 “플로우”라고 표현했던 사용 흐름은 기존 “케이스 관리” 기능으로 관리하는 방향이 맞음을 확인함.
- 메인 화면에서는 케이스 관리와 지플릿이 섞이지 않고 구분되어 보이는 것이 중요하며, 케이스/지플릿의 노출 순서를 조절할 수 있으면 충분하다고 정리함.
- 김나연은 케이스 관리/지플릿 노출 순서 관련 수정 사항을 일요일까지, 늦어지면 월요일 저녁까지 반영해 보기로 함.
- 이희정은 해당 수정이 완료될 때까지 케이스 관리 DB 입력 작업을 멈추고 기다리기로 함.
- 향후 운영 단계에서 계속 발생할 수정/개선 요청에 대응하기 위한 별도 AI 기반 유지보수 서비스(Rise OS 성격)에 대해서도 설명하고 관심 여부를 확인함.
- 이희정은 지니아 프로젝트가 단순 MVP가 아니라 본인의 10년 노하우, 유사나 조직 운영 경험, 자녀에게 물려주고 싶은 지식/가치관을 구조화하는 장기 로드맵의 출발점이라고 설명함.

## 결정사항

1. 유튜브 링크 노출 방식 수정
   - 현재 유튜브 DB가 간단 요약 중심으로 불러와지는 문제를 확인.
   - 링크 자체가 하이퍼링크 형태로 보이고 클릭 시 이동 가능하도록 수정 예정.

2. 유튜브 자막/스크립트 처리 원칙
   - 자동 자막 추출은 모든 케이스 대응까지 고도화하지 않음.
   - 자동으로 자막을 못 불러오는 영상은 다른 서비스로 자막을 추출한 뒤, 스크립트 DB에 별도 입력하는 방식으로 운영.
   - 스크립트 DB는 사용자에게 자막을 그대로 보여주기 위한 용도라기보다 검색/참조용으로 활용.

3. 케이스 관리와 지플릿의 역할 구분
   - 지플릿: 개별 기능/DB 참조 단위.
   - 케이스 관리: 특정 상황/업무 흐름에 맞춰 필요한 지플릿과 수집 항목을 묶어 제공하는 단위.
   - 이희정이 요청한 “소비자 관리 순서/목차/가이드” 성격은 지플릿을 추가 생성하는 것보다 케이스 관리에 넣는 것이 적합함.

4. 메인 화면 노출 방식
   - 케이스 관리와 지플릿이 한 화면에 뒤섞여 보이지 않는 것이 중요.
   - 케이스 관리는 지플릿 위쪽 별도 영역에 표시되는 구조로 이해.
   - 노출 순서를 조절할 수 있게 하는 정도면 현재 요청은 충족 가능하다고 정리.

5. 작업 일정
   - 김나연이 케이스 관리/지플릿 노출 순서 관련 수정 사항을 확인하고 반영.
   - 목표: 일요일까지, 늦어질 경우 월요일 저녁까지.
   - 이희정은 그 전까지 케이스 관리 DB 입력을 중단.
